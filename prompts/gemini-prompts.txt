This sounds like a highly practical, fun, and slightly intense interview assignment! Intentface is a GenAI-focused company, so they are definitely looking for how you handle context, tool use (function calling), and agentic behavior rather than just how pretty you can make a frontend.

Since you only have 4 hours, you need to be extremely strategic. Here is a breakdown of how to tackle this, along with a gentle challenge to your proposed tech stack.

1. The Stack: Rethinking React
You mentioned wanting to use React + Node Express. While a web UI is nice, the instructions specifically mention: "Periodical updates on stdio".

stdio means Standard Input/Output—the terminal.

If you spend 2 hours setting up Vite, React, Express, CORS, and a chat UI library, you will burn half your time on boilerplate.

My Recommendation: Build a pure Node.js (or Python) CLI App.

Why: It natively satisfies the stdio requirement, requires zero frontend boilerplate, and lets you focus 100% on the core AI agent logic (which is what an AI consultancy actually cares about).

Stack: Node.js (with TypeScript if you are fast with it), the official OpenAI SDK (or Anthropic SDK), and a simple local JSON file for database persistence.

2. The Architecture (The "Agent" Pattern)
To make this work in natural language, your LLM needs to act as an agent with access to tools (Function Calling).

The Loop: User types in terminal -> Message sent to LLM -> LLM decides if it needs to call a tool (e.g., add_todo, get_weather) or respond -> App executes tool -> Result sent back to LLM -> LLM responds to user.

Storage: Just use a local todos.json file. Don't waste time spinning up PostgreSQL or MongoDB. Read/write to a local file using fs.promises.

Weather: For the clothing suggestion, use a free API like Open-Meteo (requires no API keys) based on the user's location (e.g., Helsinki).

The Proposed Tech Stack
Instead of standard Express + React, a TypeScript CLI / Daemon architecture will directly satisfy the stdio requirement and prevent boilerplate bloat.

1. Core Runtime: Node.js + TypeScript
Why: TypeScript gives your LLM agent strict type safety for its tools, and Claude Code excels at generating typed code.

Execution: Run it natively via tsx (TypeScript Execute) so you don't have to fiddle with tsconfig.json build steps during a timed test.

2. Agent Framework: Vercel AI SDK (ai)
Why: The Vercel AI SDK is the current industry standard for building text, structured outputs, and agentic tool-calling workflows in the JavaScript/TypeScript ecosystem.

Key Feature: The ai core package provides a generateText or streamText function with native tools configuration, handling the entire LLM loop (call tool -> get result -> feed back to LLM) automatically under the hood. It supports Anthropic, OpenAI, or any major provider seamlessly.

3. Persistence: Local SQLite or JSON
Why: The prompt requires TODOs to persist across restarts.

Option A (Simplest): A simple todos.json handled by fs/promises.

Option B (Slightly cleaner for schema scalability): better-sqlite3 or a micro-ORM like Drizzle targeting a local SQLite file. Claude Code can write this in seconds, and it looks a bit more professional for a Senior/Mid-level interview.

4. External Data: Open-Meteo API
Why: It provides global weather forecasts entirely via open HTTP GET requests—no API keys, no environment variables, no registration required. It's the ultimate time-saver for assignments.
